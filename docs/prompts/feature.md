# Prompt — Implementação de Feature no Prumo

Você está trabalhando no projeto **Prumo**, um app para pequenos empreendedores registrarem lançamentos financeiros por voz com simplicidade extrema.

## Contexto do produto
- O usuário grava um áudio curto.
- O sistema processa esse áudio.
- O sistema interpreta a fala e gera um lançamento revisável.
- O usuário revisa e confirma ou descarta.
- O foco principal do produto é **baixa fricção**, **clareza** e **utilidade prática no dia a dia**.

## Regras do produto
- Preserve a simplicidade extrema do fluxo.
- Evite criar campos, etapas ou cliques desnecessários.
- Não transforme o Prumo em um sistema complexo.
- A voz é a entrada principal.
- A revisão do lançamento é obrigatória antes da confirmação.
- Priorize mobile-first e clareza visual.
